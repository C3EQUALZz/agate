# Конфигурация

**Сегодня** Agate настраивается через переменные окружения, читаемые при запуске
`agate-server`. Параллельно проектируется **файловая конфигурация `agate.toml`**;
разделы ниже чётко помечают, что есть сейчас, а что в планах.

## Переменные окружения (сегодня)

Они читаются один раз при старте процесса. Отсутствие обязательной переменной
прерывает запуск (быстрый отказ при неверной настройке), вместо старта в
деградированном состоянии.

### Прокси и сеть

| Переменная | Обязательна | По умолчанию | Значение |
| --- | --- | --- | --- |
| `AGENT_ENDPOINT` | **да** | — | URL вышестоящего AG-UI-агента, которому Agate перенаправляет инспектированный трафик. |
| `BIND_ADDR` | нет | `0.0.0.0:8080` | Адрес/порт, на котором Agate слушает входящий AG-UI-трафик. |

### Аудит (журнал прозрачности)

| Переменная | Обязательна | По умолчанию | Значение |
| --- | --- | --- | --- |
| `DATABASE_URL` | **да** | — | Строка подключения PostgreSQL для журнала прозрачности Меркла. Миграции выполняются при запуске. |
| `AUDIT_LOG_ID` | нет | создаётся новый журнал | UUID журнала прозрачности для добавления записей. Если не задан, создаётся новый журнал, и его id выводится в лог, чтобы вы могли закрепить его при перезапуске. |

### Политики

Переменные политик все необязательны. Если ни одна не задана, **все инструменты
разрешены, и ничего не редактируется**.

| Переменная | Формат | Значение |
| --- | --- | --- |
| `POLICY_TOOL_ALLOWLIST` | имена инструментов через запятую | Только эти инструменты могут выполняться. Взаимоисключающе со списком запрета. |
| `POLICY_TOOL_DENYLIST` | имена инструментов через запятую | Эти инструменты запрещены (используется, только когда нет списка разрешений). |
| `POLICY_REDACT_PATTERNS` | литеральные маркеры через запятую | Подстроки, редактируемые из выводимого текста до того, как он достигнет клиента. |

!!! warning "Списки разрешений и запретов взаимоисключающи"
    Задание одновременно `POLICY_TOOL_ALLOWLIST` и `POLICY_TOOL_DENYLIST`
    противоречиво (что побеждает?) и **прерывает запуск**, а не разрешается
    молча. Пустое или неверное имя инструмента также прерывает запуск, так что
    опечатка не может молча ослабить контроль.

### Пример

```bash
AGENT_ENDPOINT=http://agent:9000
DATABASE_URL=postgres://agate:agate@db:5432/agate
BIND_ADDR=0.0.0.0:8080
AUDIT_LOG_ID=3f6c0b1e-2c9a-4f1e-bb1b-7e2c0b9a1d34
POLICY_TOOL_ALLOWLIST=search,read_file,http_get
POLICY_REDACT_PATTERNS=sk-,AKIA,BEGIN PRIVATE KEY
```

## Файловая конфигурация: `agate.toml` (скоро)

!!! info "Спроектировано, но ещё не поставляется"
    Файл конфигурации TOML проектируется, чтобы развёртывания могли выразить всю
    конфигурацию в одном монтируемом, версионируемом файле вместо растущего
    списка переменных окружения. Приведённая ниже форма — это **предполагаемая
    структура**; ключи и значения по умолчанию **могут измениться** и помечены
    `TODO` там, где ещё не финализированы. Переменные окружения остаются
    поддерживаемыми и, как ожидается, будут **переопределять** значения из файла.

План — монтировать единственный `agate.toml` в контейнер:

```bash
docker run --rm \
  -v "$PWD/agate.toml:/etc/agate/agate.toml:ro" \
  -e AGATE_CONFIG=/etc/agate/agate.toml \
  agate-server
```

Предполагаемые секции (иллюстративно — **пока не полагайтесь на эти имена**):

```toml
# agate.toml — ПРЕДПОЛАГАЕМАЯ ФОРМА, может измениться (TODO: финализировать)

[proxy]
bind_addr      = "0.0.0.0:8080"
agent_endpoint = "http://agent:9000"
# fail_mode    = "closed"   # TODO: fail-open или fail-closed при ошибках политики
# limits       = { max_body_bytes = ..., max_run_seconds = ... }  # TODO: бюджеты против DoS

[audit]
database_url = "postgres://agate:agate@db:5432/agate"
# log_id     = "…"          # эквивалент AUDIT_LOG_ID
# hash_algo  = "sha256"     # TODO: хеш эпохи Меркла (sha2 / sha3 / streebog)
# signing    = { ... }      # TODO: конфигурация ключа подписанной вершины дерева

[policy]
# tools  = { mode = "allowlist", names = ["search", "read_file"] }  # либо denylist / allow_all
# redact = ["sk-", "AKIA"]
# TODO: более богатые правила (regex-редактирование, ограничения аргументов на инструмент, проверка URL)

[observability]
# log_level = "info"        # TODO
# tracing   = { ... }       # TODO: экспорт OpenTelemetry

[plugins]
# TODO: переключатели опциональных адаптеров/стратегий
# ag_ui = true              # транспортный адаптер AG-UI
# agent_llm = false         # будущий адаптер агент ↔ LLM
# gost_crypto = false       # стратегии Streebog / Kuznyechik / Magma
```

Пока `agate.toml` не поставляется, используйте переменные окружения выше. Эта
страница будет обновлена авторитетной схемой, значениями по умолчанию и
правилами приоритета, когда функция появится.
